при создании команд или изменение config.json синтоксис такой 
"Имя пораметра": "Занчение",
"Имя пораметра": ["Занчение", "Занчение"],
все пути к файлом указоваются относительно асистента например

command/off/off.cmd

config.json 
это совновная конфигурация 
command_dir пут к папке с командоми
speaker голос асистента может быть 
device где будет производится вычесление для создание голоса 
gpt_api api ключь для ChatGPT.

Саздание собстевых команд 
перейдите в папку с командоми саздайте папку с названием команды зайдите в неё саздайте фаил имя команды.json обезательно без пробелов 
есть 2 типа команд say wavp и run 
тип run позволяет запустить файлы .exe .cmd .bat 
say асистент скажет текст который будет указан в настройках команды 
wavp позволяет воспроизвисти wav фаил путь до файла указовается как в типи run 

в  имя команды.json должно быть примерно такое содержимое
{
    "text": ["слово при котором будет запускатся команда", "слово при котором будет запускатся команда"],
    "type": "тип команды run wavp или say",
    "run": "если тип run то тут указовается путь до исполняемого файла если say то пишится текст который должен проговорить асистент"
}

рекомендации 
распологайте файлы которые нужны для команды в папке с командной иначе это может вызвать ошибки

для получение пораметров из config.json в своём скрипте даный код 

from lorder import *
config = get_config()
x = config.get('y')

поеснение 
from lorder import *  импорт модуля для получение config.json
config = get_config() читаем config.json и информацию помешаем в переменую 
x = config.get('y') разбераем инфорчицию и остовляем только нужное x это переменоя где будет хроница конечные даные y название переменой в config.json
и помнете что вы можоте добовлят свои переменые в config.json
